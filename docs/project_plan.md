# Ruliology Forge Project Plan

## Positioning

Ruliology Forge should begin as a focused open-source toolkit for computational resilience in cellular automata. The first public release should emphasize reproducibility, simple installation, clear examples, and careful claims.

## v0.1 scope

The v0.1 release should include:

- Elementary Cellular Automata engine
- Perturbation operators
- Restoration metrics
- Rule-space scanning
- Plotting helpers
- CLI entry point
- Tests
- Documentation

## What to avoid in the first release

Avoid making the first release too conceptually heavy. Keep biological and morphogenetic claims framed as research motivation rather than established results.

Defer these until later:

- Large shell image/data libraries
- Blender integration
- Claims about biological regeneration
- Unpublished paper-dependent claims
- Heavy dependencies

## Suggested repository description

An open-source Python toolkit for studying computational resilience, perturbation, and restoration in cellular automata and rule-based systems.
